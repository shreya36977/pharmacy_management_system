# 🏥 Pharmacy Management System

## 📌 Project Overview
The Pharmacy Management System is a command-line based application developed to implement the **Dispense Medicine use case**. It automates the process of validating prescriptions, checking medicine availability, detecting drug interactions, updating inventory, and generating bills.

---

## 🎯 Objectives
- To simulate real-world pharmacy operations
- To implement Object-Oriented Design (OOD)
- To perform Black Box and White Box Testing
- To validate system correctness and reliability

---

## ⚙️ Features
- ✔ Prescription Validation  
- ✔ Inventory & Stock Checking  
- ✔ Drug Interaction Detection  
- ✔ Expiry Date Verification  
- ✔ Billing Generation  
- ✔ Command-Line Interface (CLI)  

---

## 🧱 System Design
The system is designed using:
- **Entity Classes**: Prescription, Medicine, Inventory, Bill  
- **Control Classes**: DispenseController, InventoryManager, BillingController, ValidationSystem  
- **Boundary**: CLI Interface  

---

## 🔁 Use Case: Dispense Medicine
**Flow:**
1. Pharmacist enters prescription details  
2. System validates prescription  
3. Checks stock availability  
4. Verifies expiry & drug interaction  
5. Updates inventory  
6. Generates bill  

---

## 🧪 Testing

### 🔹 Black Box Testing

- Techniques Used:
  - Equivalence Class Partitioning (ECP)
